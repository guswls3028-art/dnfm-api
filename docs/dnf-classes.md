# 던파 모바일 직업 매핑

`src/domains/auth/dnf-classes.ts` 의 `DNF_CLASSES` 와 동기화되는 사람용 표.

회원가입 시 던파 캡처 OCR 결과는 캐릭터 각성 단계에 따라 base / 1차각성 / 2차각성 어떤 표기든 나올 수 있다. 모든 표기를 `baseClass` 로 정규화한다.

## 출처

사용자(방장) 직접 검수 매핑. 던파 모바일 기준.
✓ 표시 = 사용자 본인 모험단(광기의 파도, 21 캐릭터) 으로 직접 검증.

## 던파 모바일 미출시 (PC 전용)

- 도적: 로그 / 쿠노이치
- 마창사: 뱅가드 / 다크 랜서
- 다크나이트
- 크리에이터

`inMobile: false` 로 표시. UI 의 직업 선택 list 에서 자동 제외.

## 검수 필요 항목

- **격투가(여) 스트라이커 vs 그래플러 카이저 헛갈림** — "도움핑=카이저, 그랩폿=카이저" 가 진짜 스트라이커 챔피언 → 카이저 라인인지 / 그래플러 라인인지 확인 필요.
- **프리스트(여) 인파이터** — 1차/2차 각성명 (현재 "성령기사 → 그랜드마스터") 확정 아님.
- **워리어 스톰브링어 vs 마법사(남) 스위프트마스터 스톰브링어** 동명 — UI 에서 group 으로 disambiguate.
- **던파 모바일 신규 직업 출시** 시 추가.

수정 시 `src/domains/auth/dnf-classes.ts` 의 `DNF_CLASSES` 배열도 함께 갱신.

## OCR 매칭 로직

`normalizeClassName(ocrText: string)`:
1. 공백/특수문자 (`·`) 제거
2. exact match (base / 1차 / 2차 / aliases 중 일치)
3. partial match (3자 이상 substring) — OCR 노이즈 보정
4. 못 찾으면 null (사용자가 수기 입력 fallback)

aliases 필드에 자주 헛다는 OCR 표기 누적해 정확도 향상.
"페월수화" 와 "폐월수화" 표기 모두 인식하는 게 그 예시.
